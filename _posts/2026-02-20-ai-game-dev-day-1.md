만들 게임 주제는 **뱀서라이크**(Vampire Survivors 라이크) 장르로 정했습니다. AI로 뱀서라이크 게임을 만드는 첫날, 개발 환경을 세팅하는 것부터 시작했고, 가장 먼저 한 일은 **Unity MCP**를 Cursor와 Claude Desktop에 연결하는 것이었습니다.

## Unity MCP를 Cursor·Claude Desktop에 추가하기

[Unity MCP](https://github.com/CoplayDev/unity-mcp)는 Claude, Cursor 같은 AI 어시스턴트가 Unity 에디터와 직접 소통할 수 있게 해주는 MCP(Model Context Protocol) 브릿지입니다. 에셋 관리, 씬 제어, 스크립트 편집, 반복 작업 자동화 등을 LLM에게 맡길 수 있게 해줍니다.

### 세팅 절차

1. **Unity 패키지 설치**  
   `Window > Package Manager > + > Add package from git URL...`에서 아래 주소 입력:
   ```
   https://github.com/CoplayDev/unity-mcp.git?path=/MCPForUnity#beta
   ```
2. **MCP 서버 실행**  
   Unity에서 `Window > MCP for Unity` 창을 열고 **Start Server** 클릭 (로컬 HTTP 서버가 `localhost:8080`에서 실행됨)
3. **Cursor MCP 설정**  
   MCP 클라이언트 설정 파일에 다음을 추가:
   ```json
   "mcpServers": {
     "unityMCP": {
       "url": "http://localhost:8080/mcp"
     }
   }
   ```
4. **연결 확인**  
   Cursor 설정에서 MCP 옵션을 켜고, Unity 창에서 🟢 "Connected ✓" 표시가 뜨면 준비 완료입니다.  
   **Claude Desktop**에도 같은 `mcpServers` 설정을 추가해 두었고, 둘 다 정상적으로 연결됩니다.

이제 Cursor나 Claude Desktop에서 "빨간, 파란, 노란 큐브를 만들어줘" 같은 자연어로 Unity 작업을 지시할 수 있습니다.

## Claude 프로젝트 & PM 프롬프트 설정

2D 로그라이크 게임 개발용 **Claude 프로젝트**를 만들고, 프로젝트 지침(Project Guidelines)에 **1인 인디 모바일 게임 개발자용 PM 어시스턴트** 프롬프트를 넣었습니다.

![Claude 프로젝트 지침 설정 - PM 프롬프트](/assets/images/claude-pm-guidelines.png)

PM 프롬프트 핵심 내용은 다음과 같습니다.

- **역할**: 기획·개발·아트·QA를 혼자 담당하는 개발자가 집중력을 유지하고 프로젝트를 완주할 수 있도록 돕는 것. 과도한 문서화보다 실질적인 진척과 완성에 초점
- **프로젝트 컨텍스트**: 게임명, 장르/핵심 루프, 타겟 플랫폼, 개발 엔진, 현재 단계, 목표 출시일, 현재 고민 — 매 대화 시작 시 업데이트
- **핵심 원칙**: 완성 > 완벽, 범위 크리프 경계, 주간 단위 리듬, 에너지 관리
- **PM 역할 상세**: 로드맵 관리(MVP → 콘텐츠 완성 → 폴리싱/출시), 태스크·이슈 트래킹, 주간 회고, 가벼운 문서화(One-Page GDD, 기능 스펙 메모, 아이디어 백로그 등)

주간 계획·GDD·기능 스펙·아이디어 백로그·리스크 메모 템플릿도 포함해 두었고, "이번 주 계획 세워줘", "GDD 작성 도와줘", "이번 주 회고 같이 해줘" 같은 요청으로 바로 활용할 수 있습니다.


## Claude AI Agent 스킬 구성

Claude에서 아래 네 가지 프롬프트를 받아 AI Agent에 적용할 **스킬(Skill)**을 만들어 두었습니다.

| 스킬 | 역할 |
|------|------|
| **인디 픽셀아트 게임 디자이너** | UI/UX, 캐릭터 아트 디렉션, 레벨 디자인, 게임플레이 시스템 — 1인 개발 리소스 한계를 고려한 현실적 제안 |
| **Unity 2D 게임 개발** | C# 코드 작성·리뷰, 게임 디자인·기획, 아트 에셋 가이드, 버그 디버깅 — Unity LTS 기준 실행 가능한 지원 |
| **1인 개발자 전용 게임 기획자** | Core Loop 설계, MVP 스코프 관리, 밸런싱, 수치 설계 — 혼자 기획·개발·출시까지 염두에 둔 파트너 |
| **게임 QA 전문가** | 버그 리포트 템플릿, 테스트 케이스 설계, 밸런스 검토, 릴리즈 체크리스트 — 품질 관리 전반 지원 |

기획·디자인·개발·QA를 각각 전담하는 AI 역할을 두고, 상황에 맞게 꺼내 쓸 수 있도록 세팅해 두었습니다.

## Agent Team 통합 시스템 프롬프트

위 네 가지 역할을 **한 대화 안에서 자동 전환**하도록, **1인 인디 모바일 게임 개발 — Agent Team 통합 시스템 프롬프트**를 만들어 Claude AI에 넣어 돌렸습니다. 이 프롬프트는 "올인원 개발 파트너" 한 명이 요청 키워드(GDD·기획서·픽셀아트·C# 코드·테스트·주간 계획 등)에 따라 PM·게임 기획자·게임 디자이너·Unity 개발자·QA 엔지니어 다섯 역할을 스스로 골라 응답하게 합니다. 완성 > 완벽, 범위 크리프 경계, 빠른 결정 후 검증, 에너지 관리, 실행 가능한 산출물 우선이라는 공통 철학을 두고, 역할별로 태스크 형식·주간 계획/회고 템플릿·버그 리포트/테스트 케이스 템플릿·밸런스 검토 관점 등을 갖추었습니다. **이 통합 프롬프트를 Claude AI로 돌리면서** 기획서 작성, 로드맵·마일스톤 정리, GDD 기반 구현 방향 설정까지 이어 붙여, 실제로 게임을 생성(개발)하는 흐름을 만들었습니다.

## Claude Code 에이전트 팀 구성

흐름에서 **[디자이너 + 개발자] 병렬 작업**을 실제로 돌리기 위해 [Claude Code의 에이전트 팀(Agent Teams)](https://code.claude.com/docs/ko/agent-teams) 기능도 구성해 두었습니다. 여러 Claude Code 인스턴스가 팀으로 움직이도록 조율하는 기능으로, 공유 작업 목록·에이전트 간 메시징·팀 리더에 의한 할당/종합을 지원합니다.

- **팀 리더**: 작업을 나누고, 팀원을 생성하고, 결과를 종합하는 메인 세션
- **팀원**: 각자 할당된 작업만 담당하는 별도 Claude Code 인스턴스. 서로 직접 메시지를 주고받을 수 있음
- **공유 작업 목록**: 팀 전체가 보는 작업 목록. 대기 중 → 진행 중 → 완료 상태와 작업 간 종속 관계를 관리

에이전트 팀은 **실험 기능**이라 기본 비활성화되어 있고, `settings.json`의 `env`에 `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS: "1"`을 넣거나 셸 환경 변수로 설정해야 사용할 수 있습니다. 병렬 탐색이 도움이 되는 작업(연구·검토, 서로 다른 모듈 담당, 경쟁 가설로 디버깅, 프론트/백/테스트 등 교차 계층 작업)에 적합하고, 단순 순차 작업이나 같은 파일을 여러 명이 고치는 경우에는 단일 세션이나 subagents가 더 낫다고 문서에 안내되어 있습니다. 디자이너·개발자 역할을 팀원으로 두고 병렬로 진행할 때 이 구성을 쓰면 됩니다.

## 기획서 작성 및 에이전트 팀 공유

PM이 만든 프로젝트 계획을 바탕으로 **게임 기획자 Agent** 역할로 Claude에 **2D 자동 사냥 로그라이크 게임 기획서(GDD)** 작성을 요청했습니다. Vampire Survivors 라이크, 10분 세션, 무기 6종·적 5종·스테이지 3개·보스 1종 등 MVP 범위를 정한 뒤, 핵심 루프·플레이어/무기/적/보스 시스템·스테이지 설계·레벨업·메타 성장·경제·UI/UX·수익화·밸런싱 수치 테이블·기술 설계 가이드·MVP 기능 분류·마일스톤 연동까지 담은 본격적인 GDD가 나왔습니다.

이 **기획서 문서를 에이전트 팀(Team Agent)에게 넣어 두었습니다.** 팀원들이 `CLAUDE.md`나 프로젝트 컨텍스트를 통해 같은 GDD를 참조할 수 있게 해 두면, 디자이너·개발자·QA가 각자 작업할 때 공통 기준으로 "무엇을 만드는지"를 맞출 수 있습니다. 앞으로 구현과 밸런싱은 이 GDD를 기준으로, 에이전트 팀이 병렬로 진행할 예정입니다.

## 게임 개발 흐름

이번에 구성한 환경으로 진행하려는 개발 흐름은 다음과 같습니다. 각 단계마다 사용자 검토 포인트(⏸️)를 두고, 승인 시 다음 단계로 넘어가거나 수정 요청 시 해당 Agent에서 다시 작업하도록 하는 구조입니다.

```
[PM Agent] 결과 생성
    ↓
⏸️ [사용자 검토] ← 승인 / 수정 요청
    ↓
[기획자 Agent] 결과 생성
    ↓
⏸️ [사용자 검토] ← 승인 / 수정 요청
    ↓
[디자이너 + 개발자] 병렬 작업
    ↓
⏸️ [사용자 검토] ← 승인 / 수정 요청
    ↓
[QA 루프] 자동 진행
    ↓
⏸️ [최종 검토]
    ↓
  완성 🎮
```

PM → 기획자 → 디자이너·개발자 병렬 → QA 루프 순으로 이어지고, 각 단계에서 사용자가 최종 검증 역할을 맡는 흐름입니다. 앞으로 이 환경을 활용해서 뱀서라이크 게임 개발을 진행해보겠습니다.

위에서 정리한 **《1인 인디 모바일 게임 개발 — Agent Team 통합 시스템 프롬프트》**를 Claude AI에 넣어 돌리면서 게임 개발을 진행하고 있습니다. 요청할 때마다 PM·게임 기획자·게임 디자이너·Unity 개발자·QA 엔지니어 역할이 자동으로 전환되도록 해 두었고, GDD·주간 계획·코드·테스트 케이스까지 이 프롬프트 한 흐름으로 이어 붙여 나가고 있습니다.
