Anthropic이 6월 9일 **Claude Fable 5**를 내놨다. Claude 5 패밀리의 첫 모델이자, Opus 위에 새로 생긴 **Mythos급 티어**의 모델이다. Fable 5와 Claude Mythos 5는 같은 모델을 공유하는데, Fable은 일반 공개용으로 추가 안전장치가 붙은 버전이고 Mythos는 승인된 조직에만 제공된다.

출시 조건도 특이하다. **6월 22일까지는 Pro/Max/Team 구독에 무료 포함**되고, 6월 23일부터는 구독에서 빠져 사용 크레딧(입력 $10 / 출력 $50, 1M 토큰 기준) 방식으로 전환된다. 용량이 확보되면 다시 구독 표준 기능으로 복원할 계획이라는데 날짜는 미정이다. 수요 예측이 어려워 단계적으로 푸는 모양새다.

마침 무료 기간이라 직접 굴려봤다. Opus 4.8과의 차이는 체감으로, 타사 모델과의 비교는 공개 벤치마크로 나눠서 정리해봤다. 미리 말해두자면 GPT-5.5·Gemini·Grok은 내가 직접 써본 게 아니라 공개 자료 기준이다 — 나는 Fable과 Opus 같은 Claude 계열만 써봤다.

> 미리 밝혀둘 것: 아래 수치는 출시 직후 각 사 발표 자료와 독립 평가를 모은 것이다. 비교 대상 모델들은 출시 시점도(Fable 5는 6월 9일, GPT-5.5는 4월 23일, Gemini 3.1 Pro는 2월 19일) 테스트 환경도 달라서 완전히 깔끔한 1:1 비교는 아니다. 숫자는 방향을 보는 참고치다.

## vs Opus 4.8 — 격차가 가장 큰 곳은 에이전트형 코딩

Anthropic이 공개한 모든 벤치마크에서 Fable 5가 Opus 4.8을 앞섰다. 영역별로 보면:

| 영역 | Fable 5 | Opus 4.8 | 차이 |
|------|---------|----------|------|
| SWE-bench Pro | 80.0% | 69.2% | +10.8p |
| SWE-bench Verified | 95.0% | 88.6% | +6.4p |
| FrontierCode | 약 2배 | — | 격차 최대 |

코딩 점수보다 흥미로운 건 **장기 자율 작업**이다. 파일 기반 지속 메모리를 줬을 때 Fable 5는 같은 설정의 Opus 4.8보다 성능 향상 폭이 **3배** 컸다. 이전 상태를 쌓아가며 며칠 단위로 이어지는 에이전트 작업에서 차이가 두드러진다는 뜻이다.

비전 능력도 눈에 띈다. 지도나 상태 정보 없이 게임 스크린샷만으로 포켓몬 파이어레드를 완료했는데, 이전 모델들은 같은 과제에 추가 보조 장치가 필요했다. 스크린샷에서 웹앱 소스코드를 복원하거나 과학 도표에서 수치를 추출하는 것도 가능하다.

지식·분석 쪽에서는 Hex 분석 벤치마크 90% 첫 돌파, Hebbia 금융 벤치마크 시니어급 금융 추론 부문 역대 최고 점수를 기록했다. 독립 평가에서도 Every팀이 실제 시니어 엔지니어 채용에 쓰는 테스트에서 91/100으로 Opus 4.8 대비 큰 폭의 상승을 보였다.

그리고 점수만 높은 게 아니라 **더 적은 턴으로 25~30% 빠르게** 작업을 끝낸다. 트레이드오프는 둘이다. 가격이 Opus 4.8($5/$25)의 정확히 2배고, 사이버보안·생물·화학 같은 보호 영역 요청은 Opus 4.8로 폴백되기 때문에 그 영역에서는 두 모델이 사실상 동일하다.

## 직접 써본 체감 — 깔끔해졌고, 무엇보다 빨라졌다

벤치마크 표는 표고, 무료 기간에 실제 작업에 붙여본 체감을 적는다.

가장 먼저 느낀 건 **결과물의 정돈됨**이다. Opus도 결과 자체는 좋았지만, Fable은 같은 요청에 대해 군더더기가 적고 구조가 깔끔하게 잡힌 결과물을 내놓는다. 코드든 문서든 "이걸 다시 다듬어야겠다"는 후속 작업이 눈에 띄게 줄었다.

두 번째는 **속도**다. 입출력 응답이 Opus보다 빨라졌고, 결과물을 생성하는 속도 자체도 빠르다. 벤치마크에서 말하는 "더 적은 턴으로 25~30% 빠르게"가 과장이 아니라는 걸 체감했다. 응답이 빨라진 것과 재작업이 줄어든 것이 겹치니, 체감 작업 속도는 단순히 빨라진 정도가 아니라 **작업 흐름 자체가 끊기지 않는 수준**으로 올라왔다.

다만 아쉬운 점도 분명했다. **토큰 소모량이 엄청나다.** 단가가 Opus의 2배라 같은 양을 작업해도 세션 한도가 2배 속도로 깎인다. 평소에는 여러 프로젝트를 한 번에 같이 돌리는 식으로 작업해왔는데, Fable로 같은 패턴을 돌렸더니 **20분 만에 세션 토큰을 다 써버렸다**. 구독 한도 안에서 쓰는 입장에서는 병렬 작업 습관을 그대로 가져가기 어렵다. Fable을 쓸 때는 무거운 작업 하나에 집중시키고, 나머지는 하위 모델로 돌리는 식으로 운용을 바꿔야 했다.

그리고 가장 아쉬운 건 **6월 22일까지밖에 못 쓴다는 점**이다. 23일부터는 구독에서 빠지고 사용 크레딧(API 토큰 결제)을 따로 사야 한다. 기존 Pro/Max 같은 plan 사용자 입장에서는 구독료를 내고 있는데도 사실상 못 쓰게 되는 셈이다. 용량이 확보되면 다시 구독에 포함시킬 계획이라고는 하는데 날짜가 미정이라, 이 작업 속도에 익숙해진 뒤에 다시 Opus로 돌아가야 한다는 게 제일 아쉽다.

정리하면 — 결과물은 더 깔끔하게, 나오는 속도는 더 빠르게. 둘이 곱해지니 전체 작업 속도가 엄청나게 빨라졌다. 대신 토큰은 그만큼 빨리 녹고, 그마저도 22일까지만 구독으로 쓸 수 있다.

## vs GPT-5.5 · Gemini 3.1 Pro · Grok 4 — 교차 벤치마크

여기부터는 체감이 아니라 공개 자료 기준이다. 이 모델들은 직접 써보지 않았으니, 숫자로만 위치를 가늠해본다. 가장 깨끗한 교차 비교 지표인 SWE-bench Pro(실제 GitHub 저장소의 이슈 해결 과제) 기준:

| 모델 | SWE-bench Pro | 가격 (입력/출력, 1M 토큰) | 강점 |
|------|---------------|---------------------------|------|
| Claude Fable 5 | **80.3%** | $10 / $50 | 에이전트 코딩, 낮은 환각률 |
| Grok 4 | ~75% | $3 / $15 | 비용 대비 코딩 가성비 |
| GPT-5.5 | 58.6% | $5 / $30 | 컴퓨터 사용(GUI 조작) |
| Gemini 3.1 Pro | 54.2% | $2 / $12 | 최저가, 멀티모달·롱컨텍스트 |

프런티어 벤치마크가 보통 한 자릿수 단위로 움직이는 걸 감안하면 2위와 11점 이상 격차는 이례적이다. 난도를 올리면 격차는 더 벌어진다. **FrontierCode에서 Fable 5가 29.3%, GPT-5.5가 5.7%**로 5배 이상 차이가 난다.

코딩 밖의 지표들도 챙겨볼 만하다.

- **환각률**: 독립 평가(AA-Omniscience)에서 Claude 계열은 약 36%로, GPT-5.5(약 85%)보다 훨씬 낮았다. 틀린 답의 비용이 큰 작업이면 이 지표가 결정적이다.
- **컴퓨터 사용**: 브라우저·GUI 조작은 GPT-5.5가 앞선다. OSWorld Verified 78.7%로 인간 기준선(72.4%)을 넘었고 가격도 Fable의 절반이다.
- **과학 추론**: GPQA Diamond에서는 GPT-5.5(94.4%)와 Gemini 3.1 Pro(94.3%)가 거의 동률 선두다. 모든 영역에서 Fable이 1등인 건 아니다.
- **컨텍스트**: Fable 5와 Gemini 3.1 Pro 둘 다 1M 토큰급 컨텍스트 윈도우를 제공한다.

벤치마크를 읽을 때 함정 둘. 첫째, **SWE-bench Pro는 Python, Go, TypeScript, JavaScript 4개 언어 41개 저장소 기반**이라 이 언어권 밖에서는 점수가 그대로 적용된다는 보장이 없다. 둘째, 가장 화려한 일부 점수(공격적 사이버보안 평가 등)는 일반 구매가 불가능한 Mythos 5의 것이다. Fable 5 자체는 해당 영역이 차단되어 있다.

## 그래서 내가 주로 개발하면서 사용하는 것들 중에서는

내가 주로 개발하면서 사용하는 것들 — 웹/앱은 React와 Flutter, 게임은 Unity, 서버는 Next.js — 을 기준으로 벤치마크를 해석하면:

- **React/Next.js**: TS/JS가 SWE-bench Pro에 직접 포함되므로 Fable의 우위가 그대로 유효하다. 복잡한 리팩토링·아키텍처 작업은 Fable 5, 일상 코드는 훨씬 싼 Sonnet 4.6으로 내리는 조합.
- **Flutter/Dart**: 벤치마크 밖이다. Dart는 구글 언어라 Gemini가 순위보다 잘할 수 있는 영역이고 가격도 최저라, Gemini 먼저 붙여보고 막히면 Fable로 올리는 순서.
- **Unity/C#**: 전용 벤치마크가 없다. C# 학습 데이터는 풍부해 일반 추론 우위가 이어질 가능성이 높지만, Unity 엔진 특수 지식은 변수라 직접 A/B 테스트가 답이다.

비용은 토큰 단가가 아니라 **해결한 작업당 비용(cost per solved task)**으로 봐야 한다. 단가는 2배지만 더 적은 턴으로 끝내서 재시도 비용이 상쇄될 수 있다. 실제로 써보면 재작업이 줄어드는 게 체감되기 때문에, 무거운 작업 한정으로는 단가 차이가 생각보다 아프지 않다. 반대로 멈춤 규칙이 약한 워크플로에서는 하네스가 끊을 때까지 계속 작업해 비싸지니, 잘 정의된 큰 작업을 통째로 맡기는 용도가 맞다.

Claude Code에서 쓰려면 `/model fable`로 직접 선택해야 한다. 어떤 계정에서도 기본 모델이 아니고, 예전 `opusplan` 같은 자동 분배 별칭도 Fable에는 아직 없다.

## 정리

- Fable 5는 **에이전트형 코딩과 장기 자율 작업에서 전 모델 대비 압도적**이고, 격차는 난도가 올라갈수록 커진다.
- 직접 써본 체감도 벤치마크와 일치한다. **결과물은 더 깔끔하고, 생성 속도는 더 빨라서 전체 작업 속도가 크게 올라갔다.**
- 대신 **단가가 Opus의 2배라 세션 한도가 2배 속도로 깎인다**. 여러 프로젝트 병렬 작업은 한도가 금방 바닥나니, 무거운 작업 하나에 집중시키는 운용이 맞다.
- 다만 가격이 가장 비싸고, 과학 추론 등 일부 영역은 GPT-5.5·Gemini가 동률 이상이라 **만능 1등은 아니다**.
- 구독 포함은 6월 22일까지. 23일부터는 별도 크레딧 결제라 기존 plan 사용자는 사실상 못 쓰게 된다. 이 기간에 자기가 개발하면서 쓰는 것들로 직접 검증해보는 게 벤치마크 표보다 정확하다.
